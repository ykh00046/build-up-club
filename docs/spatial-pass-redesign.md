# 공간 지향 패스 — 코어 재설계 스펙

> 결정(사용자): 이산 타깃 패스 모델 → **공간 지향 패스 + 착지 경합** 모델로 전환.
> 목표: ① 주도권(원하는 공간으로 패스) ② 가혹한 이진 실패 제거(루즈볼/세컨볼)
> ③ "공을 가진 사람만 조작"이라는 보통 축구 게임 직관. 기존 5초 전환(E1)과 결합.

## 1. 핵심 모델 변화

| | 기존(이산) | 신(공간) |
|---|---|---|
| 입력 | 액션 무장 → **특정 동료 클릭** | **임의 지점 클릭**(공간으로) |
| 패스 판정 | 레인 위험 → **성공/실패 이진** | 착지점 **최근접 경합**(us/loose/opp) |
| 실패 | 가로채임 = 즉시 턴오버 | 비등하면 **루즈볼** → 세컨볼 경합(E1) |
| 시각화 | 동료별 레인 색 | **도달 범위 그라데이션**(멀수록 진함=실패↑) |
| 정확도 | 레인 risk | **패스 능력치·포지션·몸 방향**으로 산포 |

## 2. 착지 경합(resolveLanding) — 판정의 심장

패스가 착지점 L에 도착(정확도만큼 산포된 실제 지점):
1. 최근접 us(Du, GK·패서 제외) vs 최근접 opp(Do, GK 제외).
2. **claim = (1/(거리+2)) × (1 + 트레잇)** — 가까울수록·트레잇 좋을수록 강함.
   - us: pressResistance(받는 안정), opp: pace(달려듦).
3. `pUs = usClaim / (usClaim + oppClaim)`, roll = rng.
   - **us 우세** → 받아서 계속, us 새 홀더.
   - **opp 우세** → 탈취 → 전환 창(E1).
   - **비등(|du−do|<경합폭)** → **루즈볼**: us가 더 가까우면 압박받으며 받음
     (세컨볼 획득, 실패 아님 / facts.secondBalls), 아니면 전환.

→ "한 턴 뒤면 따낼 수 있는데 실패"가 사라짐. 근접도가 결과를 정함(레인 운이 아니라).

## 3. 패스 정확도/산포

```
scatter = baseScatter × (거리/refDist) × (1.2 − pass) × orientationPenalty
실제 착지 = 의도 지점 + 산포 벡터(rng)
```
- 패스 능력치↑ → 산포↓(원하는 곳에 정확히). 포지션별 pass 트레잇 차이 반영.
- 몸 방향(orientation): 정면(FACING)=낮은 페널티, 등짐(BACK)=전·측면 페널티↑.
- **도달 범위 viz**: 홀더 기준 그라데이션 — 가까운 곳 옅음(정확), 먼 곳 진함(산포·실패↑).

## 4. 하위 시스템 재유도(이산 액션 의존분)

신 모델에선 "액션 id"가 줄어드므로 아래를 **지오메트리에서 재유도**:
- **facts**: linesBroken(홀더→착지가 opp 라인 통과?), switches(좌우 폭 큰 패스),
  windowsUsed(열린 공간 착지), runs(런 지시 유지), baits(hold/carry 유지),
  situationsResolved/decisionsMade(상황·전환 유지) → 지표·정체성·정산 그대로 작동.
- **스카우팅 추천**: 액션 id → "공간 유형"으로 재매핑(배후 공간/측면 전환/발밑 리셋).
- **렌더러**: 동료별 레인 → 범위 그라데이션 + 착지 경합 시각.
- **입력**: 동료 클릭 → 지점 클릭(터치 2탭/키보드도 지점 사이클).

## 5. 로드맵 (검증 가능한 증분, 게임을 안 깨고)

- **Inc.1 — 착지 경합(resolveLanding)** ★현재: 이진 실패 → us/loose/opp 경합.
  기존 타깃 입력 유지(into_space/to_feet가 경합으로 판정). 루즈볼=세컨볼.
- **Inc.2 — 공간 입력**: 임의 지점 클릭 패스 + 정확도 산포. 동료 클릭과 병행→대체.
- **Inc.3 — 범위 그라데이션 viz**: 도달/정확도 히트맵(렌더러).
- **Inc.4 — 하위 재유도**: facts·스카우팅·정체성을 공간 모델 기준으로 정합.
- **Inc.5 — 정리**: 구 이산 타깃·레인 프리뷰 제거, 밸런스 재튜닝, 회귀 갱신.

각 증분: 헤드리스 측정 + 회귀 + 브라우저 검증 후 커밋. 실패해도 직전 증분으로 롤백.
