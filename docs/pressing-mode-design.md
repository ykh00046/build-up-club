# 압박 탈취 모드 설계

## 목적

빌드업 클럽은 지금까지 “우리가 공을 가진 상태에서 압박을 깨는 게임”에 가깝다. 압박 탈취 모드는 반대 국면을 추가한다. 상대가 공을 가진 짧은 순간에 우리 팀이 압박을 걸어 공을 빼앗고, 다시 공격으로 전환하는 모드다.

핵심 목표는 세 가지다.

- 공격만 반복하는 단조로움을 줄인다.
- 상대 AI도 빈틈과 이득을 찾는다는 느낌을 만든다.
- 플레이어가 “소유”, “상실”, “탈취”, “재공격”을 하나의 전술 루프로 이해하게 한다.

## 한 줄 요약

상대가 빌드업을 시작하면 플레이어는 `강하게 압박`, `패스길 차단`, `블록 후퇴` 중 하나를 고른다. 성공하면 높은 위치에서 공을 되찾아 즉시 공격하고, 실패하면 뒷공간이 열리거나 상대가 압박을 벗겨 시도가 끝난다.

## 기존 게임과의 관계

기존 게임 루프:

1. 우리가 공을 잡는다.
2. 패스, 운반, 기다리기로 상대 압박을 유인한다.
3. 열린 공간이나 슛 존을 만든다.
4. 성공하거나 공을 잃는다.

압박 탈취 모드가 추가하는 루프:

1. 상대가 공을 잡는다.
2. 우리 팀이 압박 위치를 잡는다.
3. 플레이어가 탈취 방식 선택을 한다.
4. 탈취 성공 시 우리 공격으로 전환한다.
5. 실패 시 상대가 압박을 벗기고 위험한 전환이 발생한다.

즉, 이 모드는 별도 미니게임이 아니라 기존 빌드업 게임의 반대 국면이다.

## 진입 조건

압박 탈취 모드는 다음 상황에서 열 수 있다.

- 플레이어가 액션바의 `압박` 버튼을 누른다.
- 공 소유권이 잠시 상대에게 넘어간 것으로 처리된다.
- 일반 패스, 운반, 슛 액션은 잠기고 상황 선택 UI가 열린다.

초기 버전에서는 플레이어가 능동적으로 `압박` 버튼을 눌러 모드를 연다. 이후 확장 버전에서는 패스 실패, 세컨볼, 상대 골킥, 경기 이벤트에서 자동으로 열 수 있다.

## 선택지

### 1. 강하게 압박

가장 공격적인 선택이다.

- 장점: 탈취 확률이 가장 높다.
- 단점: 실패하면 뒷공간 노출이 크다.
- 성공 결과: 가장 가까운 우리 선수가 공을 되찾고 즉시 공격한다.
- 실패 결과: `압박 실패` 결과로 시도가 끝난다.

사용 감각:

> “지금 뺏으면 바로 찬스인데, 벗겨지면 끝난다.”

### 2. 패스길 차단

상대 볼 소유자를 직접 덮치기보다 다음 패스 경로를 읽는 선택이다.

- 장점: 실패 위험이 강한 압박보다 낮다.
- 단점: 탈취 확률도 약간 낮다.
- 성공 결과: 패스길을 끊고 짧은 역공으로 전환한다.
- 실패 결과: 상대가 첫 패스로 압박 밖을 빠져나간다.

사용 감각:

> “상대가 어디로 나갈지 읽고 길목에서 끊는다.”

### 3. 블록 후퇴

탈취를 포기하고 안정성을 선택한다.

- 장점: 즉시 실패하지 않는다.
- 단점: 탈취 기회와 높은 위치 재공격을 포기한다.
- 결과: 우리 골키퍼 또는 후방 빌드업 상태로 리셋한다.

사용 감각:

> “무리하지 않고 다시 우리 빌드업으로 돌아간다.”

## 확률 모델

압박 성공률은 단순 랜덤이 아니라 전술 상태를 반영해야 한다.

성공률을 올리는 요소:

- 볼 근처 우리 선수 수
- 전방 라인이 `라인 밀어내기` 상태
- 중원이 `라인 사이`를 잡고 있는 상태
- 모멘텀이 높은 상태
- 상대 볼 소유자가 압박 저항이 낮은 선수인 경우

성공률을 낮추는 요소:

- 피로가 높은 상태
- 근처 압박 인원이 부족한 상태
- 상대 볼 소유자의 패스, 침착성, 압박 저항이 높은 경우
- 같은 압박 선택을 반복해서 상대가 읽은 경우

초기 권장 범위:

- 강하게 압박: 12-74%
- 패스길 차단: 10-68%
- 블록 후퇴: 확률 없음, 안전 리셋

## AI 매칭 관점

이 모드의 핵심은 “우리도 상대도 자동으로 이득과 헛점을 찾는다”는 느낌이다.

우리 AI가 보는 것:

- 볼 근처에 몇 명이 압박 가능한가
- 어느 선수가 가장 먼저 덮칠 수 있는가
- 차단할 패스길이 있는가
- 성공 시 어느 국면에서 공격을 재개하는가

상대 AI가 보는 것:

- 우리가 전방을 너무 올렸는가
- 우리 압박이 한쪽으로 쏠렸는가
- 반복적으로 강한 압박만 쓰는가
- 압박 실패 시 어느 뒷공간이 열리는가

플레이어에게 보여줄 문장은 숫자만 나열하지 않는다.

좋은 표시 예:

- `강하게 압박: 탈취 58% · 실패 시 뒷공간 노출`
- `패스길 차단: 탈취 44% · 실패 위험 낮음`
- `블록 후퇴: 탈취 포기 · 우리 빌드업 재개`

나쁜 표시 예:

- `press = 0.58`
- `AI score high`
- `위험함`

## UI 흐름

액션바:

- 기존 액션: 운반, 패스, 기다리기, 슈팅
- 추가 액션: 압박
- 단축키: `5`

압박 버튼을 누르면:

1. 액션바 일반 버튼이 흐려진다.
2. 상황 스트립에 `압박 탈취`가 뜬다.
3. 하단 actionbar에도 선택지 버튼이 복제된다.
4. 플레이어는 `강하게 압박`, `패스길 차단`, `블록 후퇴` 중 하나를 누른다.

HUD 목표 문구:

- 압박 선택 전: `상대 첫 패스 전에 압박 방향을 결정하세요.`
- 상황 선택 중: `압박 탈취 — 지금 대응하세요`
- 성공 후: 기존 빌드업 또는 전진 목표로 복귀

## 결과와 리포트

성공 시 기록:

- `defensivePressWins += 1`
- `situationsResolved += 1`
- `decisionsMade += 1`
- 압박 성공 로그 추가
- 우리 소유로 전환

실패 시 결과:

- outcome kind: `press_broken`
- headline: `압박 실패 — 상대가 빠져나갔습니다`
- 리포트에서 수비 전환 노출 또는 압박 실패 원인을 설명

리포트에 들어갈 문장:

- `압박 탈취 1회로 상대 빌드업을 바로 끊었습니다.`
- `압박 탈취 1회 성공 — 상대 소유를 우리 공격으로 전환`

## 밸런스 원칙

압박 탈취는 항상 이득이면 안 된다. 이 모드가 의미 있으려면 세 선택지가 모두 살아 있어야 한다.

- 강하게 압박은 성공 보상이 커야 한다.
- 패스길 차단은 안정적인 중간 선택이어야 한다.
- 블록 후퇴는 재미없지만 합리적인 선택이어야 한다.

실패가 너무 약하면 플레이어는 매번 강한 압박만 누른다. 실패가 너무 강하면 아무도 압박을 누르지 않는다. 따라서 초반에는 실패를 즉시 실점이 아니라 “시도 종료”로 처리하고, 커리어 고난도나 상위 디비전에서만 실점 리스크를 키우는 편이 좋다.

## MVP 구현 범위

MVP에 포함:

- 액션바 `압박` 버튼
- `PRESSING` 국면
- 상대 볼 소유자 지정
- `강하게 압박`, `패스길 차단`, `블록 후퇴` 선택지
- 성공 시 우리 소유 전환
- 실패 시 `press_broken` 결과
- HUD/상황 스트립 표시
- 단위 테스트와 브라우저 스모크

MVP에서 제외:

- 상대가 직접 패스 코스를 애니메이션으로 전개하는 장면
- 풀 11대11 수비 미니게임
- 수동 선수 전환과 직접 태클 조작
- 압박 트랩 위치를 캔버스에 별도 그리는 고급 시각화

## 현재 반영 상태

현재 코드에는 1차 구현 일부가 들어가 있다.

반영됨:

- `PRESSING` 국면
- `engine.openPressingMode()`
- `defensivePress` 상태
- `강하게 압박`, `패스길 차단`, `블록 후퇴` 선택 처리
- `press_broken` 결과 타입
- 액션바 `압박` 버튼
- HUD의 압박 국면 목표 문구
- 압박 성공, 압박 실패, 후퇴 리셋 단위 테스트

아직 검증 또는 후속 작업 필요:

- 브라우저 스모크에 압박 버튼과 선택지 확인 추가
- 모바일/데스크톱 시각 QA
- 전술 리포트 문구 최종 한글 표시 확인
- 압박 모드에서 캔버스 프리뷰가 과도하게 보이지 않는지 확인
- 커리어 결과 정산에서 압박 성공이 보상 루프에 잘 반영되는지 확인

## 이후 확장

1. 자동 압박 트리거
   - 패스 실패 후 세컨볼
   - 상대 골킥
   - 경기 이벤트

2. 압박 성향 훈련
   - 전방 압박 특화
   - 미드블록 차단 특화
   - 안정 후퇴 특화

3. 상대 대응 학습
   - 강한 압박 반복 시 롱볼 회피
   - 패스길 차단 반복 시 운반 탈출
   - 후퇴 반복 시 상대 점유 안정

4. 보드 시각화
   - 압박 가능 선수 하이라이트
   - 차단 가능한 패스길 표시
   - 실패 시 열리는 뒷공간 그림자

## 완료 기준

압박 탈취 모드는 다음 조건을 만족하면 1차 완료로 본다.

- 플레이어가 경기 중 `압박`을 눌러 모드를 열 수 있다.
- 선택지가 명확히 표시된다.
- 성공/실패/후퇴 결과가 서로 다르다.
- 성공하면 실제로 우리 소유가 된다.
- 실패하면 기존 결과 리포트에 압박 실패가 남는다.
- 모바일에서도 선택 버튼이 44px 이상이고 겹치지 않는다.
- 테스트가 압박 성공, 압박 실패, 후퇴 리셋을 고정한다.
