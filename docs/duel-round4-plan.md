# 듀얼 4라운드 플랜 — 진입 다양화 · hold 경제 · 종합 재감사

2026-07-03 에이전트 듀얼 1R~3R(감사 2회 + 수리 3회, dad0a46~e1545be)의 후속 계획.
세 플랜은 의존성이 있어 순서가 중요하다:

```
플랜 A (진입 다양화)  ─┐
                       ├→  플랜 C (종합 재감사 = 라운드 마감 게이트)
플랜 B (hold 경제)    ─┘
```

A와 B는 구조 변경이므로 먼저, C는 그 결과를 검증하는 게이트이므로 마지막.
A·B는 서로 독립(다른 서브시스템)이라 병행 가능하지만, 측정 오염을 피하려면
**A 완료·커밋 → B 완료·커밋 → C** 직렬을 권장.

## 플랜 A — 수비 국면 진입 다양화 (상실 지점이 의미를 갖게)

### 현재 상태 (측정 근거)

- 진입 홀더 **opp-gk@(100,34) 100%** — 4시나리오 × 3상실지점 × 150시드 전부
  (probe2-defense-entry). 어디서 뺏기든 상대는 항상 자기 GK부터 재시작한다.
- 원인: `resolvePossession('turnover')`가 최후방 빌더 고정 선택 (possession.js).
- 단순히 "상실 지점 최근접 상대 진입"으로 바꾸면 **국면이 1결정으로 붕괴**:
  회수 48.8→12.3%, 실점 7.8→14.0% (카운터팩추얼 실측, n=1200). GK 리셋은
  비현실적이지만 현재 국면 길이의 유일한 지지대다.

### 설계

**A-1. 스텝 전진 캡 (선행 조건 — 국면 길이의 새 지지대)**

- `applyOpponentBuildStep`에서 후보 레인 중 전진(progress)이 `STEP_CAP`(제안:
  22m)을 넘는 레인은 **대안이 있으면 제외** (기존 prog<1 필터와 같은 패턴,
  opp-build-policy.js `chooseOppBuild`에 배치).
- 효과: GK(100)→CB(53) 47m 원패스가 2홉으로 쪼개져, 진입 위치와 무관하게
  결정 수가 자연 확보된다. 지지대를 "GK 고정 진입"에서 "스텝 물리"로 이전.
- direct 성향의 burst 레인은 캡의 예외 후보로 남길지 결정 필요 — 남기면
  "직선 역습은 캡을 뚫는다"는 성향 질감, 빼면 일관성. **권장: direct의
  burst 레인만 캡 1.5배** (질감 우선).

**A-2. 상실 지점 기반 진입 (옵션 플래그 뒤)**

- `createEngine(..., { defenseEntry: 'loss' })` 신설. 기본값 `'reset'`(현행 GK).
- `'loss'`: 진입 홀더 = `transition.loss` 최근접 상대 (이미 possession.js의
  `press_regain`이 같은 최근접 매칭 이디엄 보유 — 재사용).
- 진입 깊이별 위험 티어가 자연 발생: 높은 곳(상대 진영)에서 뺏기면 상대도
  후방부터(현행과 유사), 우리 진영 깊은 곳에서 뺏기면 위험한 진입(1~2결정,
  전술 파울·지목의 가치 급등).
- **한 국면 최소 2결정 보장**: 진입 홀더 x가 `DEFENSE_SHOT_X + STEP_CAP`보다
  낮으면(즉시 슛권) 진입을 한 티어 뒤 선수로 승격. "0결정 즉사"는 결정 게임의
  부정 — 유일한 하드 가드.

**A-3. 밸런스 재측정 · 기본값 전환**

- probe2-defense-entry 카운터팩추얼을 A-1+A-2 조합으로 재실행.
- 수용 밴드(성향 balanced, A1 기준): 정책별 회수 30~65%, 실점 5~18%,
  결정 수 분포 2~4 (1결정 국면 비율 < 10%).
- 밴드 안이면 `defenseEntry: 'loss'`를 커리어 기본값으로. 밴드 밖이면
  STEP_CAP·티어 승격 규칙을 조정해 반복.

### 완료 기준

- [ ] 진입 홀더 분포가 상실 지점을 따라 3군데 이상으로 갈라진다.
- [ ] 1결정 국면 < 10%, 평균 결정 수 ≥ 2.2.
- [ ] 회수/실점이 수용 밴드 내, 5택 전부 생존 유지.
- [ ] defenseLoop:false·'reset' 경로 회귀 무결(기존 테스트 계약 유지).

### 리스크

- 스텝 캡이 상대 빌드업을 너무 느리게 만들면 수비 국면이 루즈해짐 —
  DEFENSE_MAX_STEPS(3)와 캡의 조합을 EV 프로브로 확인.
- 지휘자·성향 레인과 삼중 상호작용 — 변경 후 probe2-defense-commander2 재실행 필수.

## 플랜 B — hold/압박 게이지 경제 심화 ("정지 박자"의 실체화)

### 현재 상태 (측정 근거)

- 게이지: 시작 22, **중앙값 2, p90 22, max 54** (자연 상승 +2/액션 도입 후).
- hold: 보드 후보·정책 분기 존재하나 **AI 자기대국 사용률 ~0%** — 효율적
  플레이는 게이지가 차기 전에 라인을 뚫는다(유출 우세: 페이즈 전환 -10/-6,
  라인브레이크 -6/개, 창사용 -8, pass_space -2).
- 베이트 0.00회/경기 → resolveScoreline의 isolation 보너스(baits≥2 && switch)가
  영구 사문 — 경제 죽음이 정산 보상까지 잠근다.

### 설계 원칙

hold를 억지로 좋게 만들지 않는다(스팸 재발 — carry 초안 goal 36% 붕괴의 교훈).
대신 **게이지가 실제로 오르내리는 밴드**를 만들어 hold의 니치(고압박·레인 폐쇄)가
경기당 1~2회 자연 발생하게 한다.

**B-1. 유출 항 재조정 (파라미터 스윕)**

- 후보 다이얼: 라인브레이크 -6 → -3 (가장 빈번한 유출, 경기당 ~4회),
  창사용 -8 유지(명시적 보상 순간), 페이즈 전환 -10/-6 유지(마일스톤).
- 스윕 스크립트(`scripts/probe4-gauge-sweep.mjs`): 다이얼 3~4조합 × 300경기로
  게이지 중앙값·hold 후보 노출 횟수·goal%를 한 번에 표로.
- **주의**: 게이지는 상대 커밋 확률(press.js decide()의 `0.6+p/100*0.8`)에도
  들어간다 — 게이지 밴드가 오르면 상대 압박도 세진다(자기 균형이지만
  goal%/fail% 게이트 필수).

**B-2. hold 후보 재캘리브레이션**

- 목표 지표: hold 사용률 3~10%/경기(액션 기준 1~3%), 사용 시 창 발생률 ≥ 40%.
- B-1의 밴드가 자리 잡으면 hold 후보 문턱(reward > 0.02)과 pHat 계수(0.30)를
  실측 커밋 유도율에 맞춰 조정. 스팸 가드(연속 hold 페널티)는 유지.

**B-3. 베이트 루프 복원 확인**

- baits ≥ 1/경기 달성 시 resolveScoreline isolation 보너스가 실제로 발동하는지
  career-sim + 셀프플레이에서 확인(사문 해제).

### 완료 기준

- [ ] 게이지 중앙값 12~30 밴드, p90 ≥ 40.
- [ ] hold 사용률 1~3%(액션), 베이트 ≥ 0.3회/경기.
- [ ] goal% 19~23 유지, fail% 강도 단조 유지(press-sanity 600).
- [ ] career-sim 게이트 통과(경제 수치 변경 시 필수 — CLAUDE.md).

### 리스크

- 게이지 인플레가 pressure_surge 등 동적 상황 발동률을 왜곡할 수 있음 —
  events/situations 발동 빈도를 스윕 표에 함께 기록.
- 사람 플레이(비효율 빌드업)는 AI보다 게이지가 빨리 참 — AI 기준으로만
  조이면 사람에게 과압박. mid 난이도 실플레이 스모크 1회 권장.

## 플랜 C — 4R 종합 재감사 (라운드 마감 게이트)

A·B 적용 후 공수 2에이전트 듀얼 감사를 재실행한다. 이번 라운드에 추가된
메커니즘(5택·마크 감쇄·파울 예산·진입 다양화·게이지 밴드)이 대상.

### 공격 에이전트 질문지

- carry 20%/hold 신규 메타가 건강한가 — 스팸 재발, 액션 엔트로피, 셀별 분포.
- 게이지 밴드가 공격 국면 체감(커밋 빈도·창 생성)에 미친 영향.
- vhigh 붐-버스트(goal +1.9%p) 유지/악화 여부 — fail% 단조는 불변식.
- A2 26.5% vs A1 11.5% 시나리오 내 편차 — 데이터 손질 대상 식별.

### 수비 에이전트 질문지

- **5택 생존 매트릭스**: 성향 4종 × 국면 상황(스텝/beaten/contained)별로
  각 선택이 최선이 되는 셀이 존재하는가. 어느 선택도 전 상황 지배/열등 금지.
- 마크 감쇄(0.6→0.3) 악용: 마크 1회 후 cut 전환 같은 콤보가 지배 전략인가.
- 파울 예산: 정책 AI가 파울 2회를 태우는 시점 분포, 3회+ 카드 발동률,
  파울 스팸이 실점 기대값에서 실제로 손해인가.
- 진입 다양화 후 국면 길이 분포(1/2/3/4결정 비율)와 지휘자 Δ실점 재측정 —
  3결정 게임에서 지휘자가 드디어 실효를 갖는가(2R에선 Δ -0.007~+0.013 노이즈).
- 회수 위치(press=높음/cut=깊음)가 진입 다양화 후 실제 가치 차이를 만드는가.

### 방법 규약 (1R·2R과 동일)

- 에이전트 2개 병렬, 기존 파일 무수정, 신규 프로브는 `probe4-{attack|defense}-*.mjs`.
- 기존 probe/probe2/probe3 스위트는 회귀 확인용으로 우선 재실행.
- 출력: `[심각도] 제목 / 증거(수치·시드·file:line) / 제안` + 회귀 확인 표.

### 불변식 (이 라운드의 합격선)

1. 수비 5택 전부 생존 — 어떤 성향×상황 조합에서든 최선이 되는 선택이 각각 존재.
2. 성향 4티어 분리 유지(실점률 단조: safe < balanced < aggressive ≈ direct).
3. 공격 fail% 강도 단조(low<mid<high<vhigh), 타임아웃 0, 텔레포트 0.
4. 자기대국 goal 19~23%, NaN/Infinity 0.
5. career-sim 완주(1부 우승 + 프레스티지) 게이트.

## 공통 안전장치

- 모든 구조 변경은 **옵션 플래그 뒤에서 시작**(defenseEntry 등) — 기존 테스트
  계약을 깨지 않고 A/B 대조 측정 후 기본값 전환. (A-stage defenseLoop,
  possessionTurnoverLoop과 같은 패턴.)
- 커밋 단위: 플랜별 1~2커밋 + 프로브 커밋 분리(이번 라운드 관례 유지).
- 각 플랜 종료 시 게이트: 테스트 스위트 전체 + career-sim + selfplay-probe 300.
- 밸런스 다이얼 변경 이력은 커밋 메시지에 "측정 전→후" 수치를 반드시 남긴다.

## 플랜 C 감사 결과 (2026-07-03, 6dda3de)

probe4-attack-faildecomp/failsplit, probe4-defense-5choice/policyuse.

### 1순위 이상 신호 판정

- **fail% 22→10 급락 = 재라벨링, 밸런스 붕괴 아님.** defenseLoop OFF(순수 공격)는
  fail 강도 단조 유지(24→35%, vhigh 최난). ON은 수비 국면이 턴오버를 "다시 잡아
  슛까지 가는 near"로 전환 — 총 비득점률 ~80% 안정, goal% 밴드 내(18-21%),
  매치 실점 0.8-1.8%. A단계 원설계(fail 38→24)의 연장선.
- **goal% 강도 상충 해소.** 깨끗한 분해(faildecomp)에서 vhigh goal ≤ low goal
  양방향 일치(역전 없음). 이전 press-sanity 역전은 수비 국면 회수가 vhigh 골을
  부풀린 측정 잡음이었음. 백프레셔 0.79 + discipline로 램프 순방향 확정.

### 불변식 스코어카드

### 수리: 스텝 캡 26→32 (6dda3de)

캡 26이 로스터 간격(CB→MF 28m)에서 전진 가능 레인을 스텝당 1.19개로 조여
성향을 붕괴시켰다(safe=balanced 소수점 동일). 캡 32로 3.20개 회복 → 티어·지휘자
동시 부활. 대가: loss 결정 수 2.0→1.1-1.5(단, loss=위기 진입은 본디 빠름 —
reset=빌드업 진입만 ≥2.0 유지, 게이트 ≥2.2는 reset에만 적용하도록 이해 정정).

### 잔존(백로그 이관, 실해 낮음)

- drop AI 미사용 — **정리(098d24a)**: 순EV로 늘 열등이라 억지 부양 불가(초안
  실점 1.8→6.0 악화 확인). 대신 위기 에스컬레이션 사다리로 정직히 배치 —
  가망 있으면 press/cut/mark, 가망<0.4+벗겨짐+파울 예산 → 파울(리셋), 파울
  소진 → drop(데미지 컨트롤 최후 수단). base는 "안전한 탈출=압박 무의미" 후퇴
  니치 유지. drop은 실측 최저 실점(concede-minimizer) = 리드 보호용 플레이어
  선택지이며 AI가 최후 지점에서만 뽑는 게 정상(park-the-bus 성격).
- cut/mark 니치 중복 — **해결(1ba14ed)**: cut을 예측 할인 제거해 성향 독립
  존 커버로, mark를 markP×pred 직접 스케일 대인 도박으로 분리. 정책이 예측
  가능 상대엔 mark(safe/bal 48/45%), 불가엔 cut(agg/direct 36/59%)을 골라 니치
  분화. mark 성향 단조 복원(safe 85%→direct 52%).
- reset+강제성향 조합의 safe/bal/agg 동일 — 자유 플레이는 null 성향이라 실경로
  아님(비관찰).

### 치명 발견 → 수리 (323c35b)

- **loss 진입 5택 붕괴**: 4R cut/mark/drop 니치가 reset에서만 검증됐고 커리어가
  실제 쓰는 loss 진입에선 cut·mark·drop 전 성향 0%(press+foul 2택)로 붕괴.
  원인은 정책 평가식 — offLaneThreat이 loss(깊은 진입)에서 0이라 cut/mark 항이
  붕괴하고 press=regainP²+carrierRisk가 폭증. 실제 net-EV는 loss에서도 cut/mark가
  최선인데 정책만 press 과대평가. **수리**: press 제곱 제거+위험 페널티, cut 안전
  base+offLaneThreat 가산 복원, mark flat. 결과 loss에 press 34-44%/cut 49-61%/
  foul 6-18% + mark safe 49% 부활. 티어 단조 유지, 테스트 30/30.

### 잔존(백로그, 실해 낮음)

- **E1 성향 티어 비단조** — 게겐 미러(E1)에서 press/cut이 aggressive 상대에
  실점 더 낮음(safe 8.3 > aggressive 6.5). 원인은 버그 아님: cut/press가 위험
  루트(aggressive의 gamble/trap)를 잘 가로채는 설계 기능의 창발. direct(최고
  위험)는 모든 선택에서 최고 실점 유지 → 핵심 티어 신호 보존. 과튜닝(cut 반위험
  성질 nerf) 위험이라 수용. mark은 단조(safe 8.3<direct 15.9).
- **hold 후보 위치 의존 문턱** — "압박≥35" 니치가 박스 인근(depthMul 1.35)에선
  게이지 7-9에서도 노출. 단 hold 실사용 0%(EV 경쟁 패배)라 후보 목록에만 영향,
  실플레이 무해. hold 실사용은 미래 보상 모델링 필요(별도 세션).
- **isolation 정산 보너스 게이트 11.1%** — **해결(0e94839)**: 진단상 열린 약측
  스위치가 21% 가용인데 실행 23%뿐(evaluator 저평가). scanOptions에 약측 오픈
  보너스(오픈니스 비례 ×0.24) + 전환 판정 |Δy|>20→16. switches 29.9%→62.3%,
  게이트 11%→31.3%, goal% 21.5% 밴드 유지. 병목이 baits로 이동.
- **drop 커리어 미등장** — loss(위기)는 짧아 blockStall·foul 예산 축적 불가.
  drop은 긴 국면(reset)·리드 보호용 유지. crisis 사다리는 reset에서만 발동.
