# Build-Up Club 큰 그림 및 향후 플랜

## 1. 제품 비전

Build-Up Club은 “한 번의 빌드업 모먼트가 클럽의 장기 성장으로 이어지는” 전술 축구 게임이다.

핵심은 단순히 패스를 성공시키는 것이 아니라, 플레이어가 다음을 반복해서 학습하고 선택하게 만드는 것이다.

- 상대 압박 구조를 읽는다.
- 전술 행동을 선택한다.
- 경기 결과 리포트로 원인을 이해한다.
- 훈련/성장 선택으로 클럽 정체성을 만든다.
- 다음 경기에서 그 선택의 효과를 체감한다.

장기 방향은 “전술 퍼즐 + 커리어 성장 + 클럽 정체성”의 결합이다. 매 경기는 짧지만, 누적 선택은 클럽의 스타일을 만든다.

## 2. 현재 기준 상태

현재 빌드는 작동 가능한 베이스라인이다.

구현된 핵심 축:

- 11v11 전술 모먼트 엔진
- 압박, 위험도, 패턴 적응, 피로, 모멘텀
- 전술별 상대 성향 차이
- 경기 중 선택 이벤트
- 경기 후 전술 리포트
- 클럽 성장/디비전/수익/프레스티지
- 클럽 정체성 XP
- 리포트 기반 훈련 선택
- 훈련 효과의 다음 경기 반영
- 저장 데이터 마이그레이션
- Chromium 통합 스모크 및 주요 회귀 테스트

현재의 가장 큰 강점은 “전술 선택 → 결과 설명 → 성장 선택 → 다음 경기 효과”의 루프가 연결되기 시작했다는 점이다.

현재의 가장 큰 약점은 플레이어가 경기 전에 상대를 읽고 의도적으로 대응하는 UX가 아직 약하다는 점이다. 엔진에는 차이가 있지만, 플레이어가 그 차이를 충분히 예측하고 활용하기 어렵다.

## 3. 핵심 게임 루프

목표 루프:

1. 허브에서 다음 상대와 위험을 확인한다.
2. 전술 브리핑에서 상대 압박 성향과 약점을 읽는다.
3. 라인 의도와 난이도/대응 방향을 고른다.
4. 경기 중 빌드업 모먼트를 해결한다.
5. 상황 이벤트에 대응한다.
6. 결과 리포트에서 잘 통한 패턴과 읽힌 패턴을 확인한다.
7. 리포트 기반 훈련을 선택한다.
8. 클럽 정체성 XP와 다음 경기 보정이 누적된다.
9. 더 높은 디비전과 더 강한 상대를 상대한다.

이 루프에서 가장 중요한 설계 원칙은 “설명 가능한 체감”이다. 플레이어는 왜 성공했는지, 왜 실패했는지, 다음에 무엇을 바꾸면 되는지 알아야 한다.

## 4. 우선순위 로드맵

### P0. 안정성 및 개발 기반

목표: 저장 데이터와 테스트 기반을 안정화해 이후 기능 추가 비용을 낮춘다.

현재 상태:

- `saveVersion = 2` 도입
- 기존 저장 데이터 정규화
- 훈련 효과 저장/만료 검증
- 주요 회귀 테스트 통과

남은 작업:

- `.verify` 산출물과 `.bkit/runtime` 파일의 Git 추적 정책 정리
- 저장 데이터 마이그레이션 테스트를 브라우저 localStorage 레벨까지 확장
- 릴리즈 전 “새 세이브/구 세이브/깨진 세이브” 3종 스모크 추가

완료 기준:

- 기존 세이브가 새 버전에서 깨지지 않는다.
- 저장 데이터 필드 추가가 테스트 없이 들어가지 않는다.
- Git diff에 런타임 산출물이 섞이지 않는다.

### P1. 상대 스카우팅 및 사전 대응 UI

목표: 엔진 내부의 상대 성향 차이를 플레이어가 경기 전에 읽고 대응하게 만든다.

구현 방향:

- 전술 브리핑에 상대 성향 카드 추가
  - 잘 막는 행동
  - 취약한 행동
  - 위험한 반복 패턴
  - 추천 라인 의도
- 상대 scheme별 문구와 추천 액션 매핑
  - 대인 압박: 원투/써드맨 추천, 반복 발밑 패스 경고
  - 지역 블록: 빠른 전환 추천, 중앙 공간 패스 경고
  - 게겐프레스: 지연/운반 경고, 원터치 조합 추천
  - 하이브리드: 써드맨 우회 추천
- 경기 중 전술 로그와 브리핑 용어 통일

완료 기준:

- 플레이어가 경기 전 상대별 대응 전략을 이해할 수 있다.
- 브리핑 추천과 실제 엔진 보정이 일치한다.
- 테스트에서 scheme별 추천 텍스트와 위험도 factor가 함께 검증된다.

### P2. 리포트-훈련-다음 경기 체감 강화

목표: 경기 후 훈련 선택이 단순 보상이 아니라 다음 경기 전략 결정에 영향을 주게 만든다.

현재 상태:

- 훈련 선택 시 포지션 +1, 정체성 XP, 다음 경기 `trainingEffects` 저장
- 다음 경기 액션 위험도와 스코어라인 수행 보정에 반영

다음 작업:

- 결과 화면에 “다음 경기 효과”를 명확히 표시
  - 예: “다음 경기: 써드맨/원투 위험도 감소”
- 허브 효과 칩에 더 구체적인 설명 추가
- 훈련 효과가 브리핑에도 노출되도록 연결
- 훈련 선택지 간 전략적 trade-off 강화
  - 즉시 레벨 성장
  - 다음 경기 특정 액션 보정
  - 2경기 지속 수비 안정
  - 정체성 XP 집중

완료 기준:

- 플레이어가 훈련 선택 전 “다음 경기에 무엇이 바뀌는지” 알 수 있다.
- 같은 리포트라도 선택지에 따라 다음 경기 접근이 달라진다.

### P3. 시즌/디비전 목표 다양화

목표: 승격 외에도 장기 반복 플레이 목표를 제공한다.

구현 후보:

- 디비전별 미션 확장
  - 무실점 경기
  - 특정 정체성 XP 달성
  - 특정 전술 행동 성공
  - 3경기 연속 같은 정체성 유지
- 시즌 목표 카드
  - “측면 전환형 클럽으로 3연승”
  - “게겐프레스 상대에게 2승”
  - “실점 0 경기 2회”
- 목표 보상
  - 철학 포인트
  - 특수 훈련
  - 클럽 색/배지/칭호

완료 기준:

- 플레이어가 “이번 시즌은 어떤 팀을 만들지” 선택할 수 있다.
- 단순 승리 외에도 전술적 목표가 플레이 동기를 만든다.

### P4. 클럽 정체성의 장기 분기화

목표: 정체성 XP가 단순 표시를 넘어 클럽 운영 방향을 바꾸게 만든다.

구현 후보:

- 정체성 레벨
  - Lv1: 표시만
  - Lv2: 해당 훈련 등장 확률 증가
  - Lv3: 관련 액션 소폭 안정화
  - Lv4: 고유 퍽 해금
- 정체성별 고유 퍽
  - 점유 조립형: 반복 패스 적응 완화
  - 직선 전진형: 침투 성공 후 xG 증가
  - 측면 전환형: 전환 후 약측 우위 증가
  - 압박 회피형: 압박 이벤트 발생 시 선택지 강화
- 정체성 전환 비용
  - 한 스타일에 오래 투자하면 강해지지만, 다른 스타일로 전환하려면 시간이 든다.

완료 기준:

- 클럽마다 플레이 감각이 다르게 성장한다.
- 같은 상대라도 정체성에 따라 다른 해법을 갖는다.

### P5. UX/시각 피드백 고도화

목표: 전술 정보가 많아져도 읽기 쉽고, 선택 결과가 즉시 보이게 만든다.

구현 후보:

- 브리핑 카드 레이아웃 정리
- 위험도 factor 툴팁 개선
- 훈련 효과/정체성 XP 애니메이션
- 모바일 하단 액션바 정리
- 경기 중 상황 배너의 선택 결과 시각화
- UI 감사 스크립트 안정화

완료 기준:

- 데스크톱/모바일 모두 핵심 선택이 한 화면에서 이해된다.
- 스크린샷 기반 UI 감사가 안정적으로 실행된다.

## 5. 추천 작업 순서

다음 4개 스프린트로 나누는 것이 적절하다.

### Sprint 1: 상대 스카우팅 UI

범위:

- scheme별 브리핑 카드
- 추천/주의 액션 표시
- 브리핑과 엔진 factor 매핑 테스트

이유:

- 현재 엔진 차이를 플레이어가 인지하기 어렵다.
- 이 작업이 들어가야 전술 선택이 “찍기”가 아니라 “판단”이 된다.

### Sprint 2: 훈련 효과 UX 강화

범위:

- 결과 화면 선택지에 다음 경기 효과 명시
- 허브/브리핑에 활성 훈련 효과 표시
- 훈련 trade-off 조정

이유:

- 이미 시스템은 연결됐다.
- 이제 플레이어가 그 연결을 명확히 볼 수 있어야 한다.

### Sprint 3: 시즌 목표와 정체성 목표

범위:

- 시즌 목표 카드
- 정체성 기반 목표
- 보상 설계

이유:

- 반복 플레이 동기를 강화한다.
- 승격만으로는 중후반 목표 밀도가 부족하다.

### Sprint 4: 정체성 퍽

범위:

- 정체성 레벨
- 정체성별 고유 퍽
- 밸런스 테스트

이유:

- 게임의 장기 차별성을 만든다.
- 너무 빨리 넣으면 밸런스가 흔들리므로 스카우팅/훈련 UX 이후가 적절하다.

## 6. 리스크와 대응

### 리스크 1: 시스템이 많아져 플레이어가 부담을 느낄 수 있음

대응:

- 브리핑은 “추천 1개 + 주의 1개 + 이유 1개” 수준으로 제한한다.
- 상세 factor는 펼침/툴팁으로 숨긴다.

### 리스크 2: 훈련 효과가 너무 강하면 난이도가 급락함

대응:

- 액션 위험도 보정은 0.84~0.90 범위 유지
- 스코어 보정은 `execAdd 0.03~0.04` 수준 유지
- 1경기 지속을 기본으로 하고, 강한 효과만 2경기 지속

### 리스크 3: 정체성 보너스가 최적해를 고정할 수 있음

대응:

- 상대 scheme별로 유리한 정체성이 달라지게 설계한다.
- 한 정체성이 모든 상대에게 정답이 되지 않게 한다.

### 리스크 4: 저장 데이터 변경 누락

대응:

- 신규 persistent field 추가 시 `freshState`, `normalizeState`, 테스트를 함께 수정한다.
- `identity-training-test` 또는 별도 migration test에 케이스를 추가한다.

## 7. 다음 액션

즉시 다음 작업은 P1 “상대 스카우팅 및 사전 대응 UI”다.

구체 작업:

1. `scenarios.js` 또는 별도 데이터에 scheme별 scouting metadata 추가
2. 전술 브리핑 overlay에 scouting card 추가
3. 추천/주의 액션과 실제 `tacticalFactors` 매핑 일치 테스트 추가
4. Chromium smoke에서 브리핑 카드 노출 검증

이 작업이 끝나면 플레이어는 “상대를 보고 전술을 고르는” 단계로 올라간다.
