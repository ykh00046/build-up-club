# 빌드업 클럽 — 발전 방향 고찰과 플랜 (2026-07-07)

## 0. 지금 서 있는 곳 — 정직한 진단

세 프로젝트의 통합(전술 코어 + 클럽 메타 + 실시간 감각)으로 출발한 이 게임은,
8라운드의 적대 감사로 **턴제 결정 코어의 밸런스를 측정 기반으로 완성**했고(게이트
32종, selfplay ~23.5%, concede 밴드 7/10), 이번 주 **"흐르는 시간 속의 결정"이라는
새 정체성**으로 전환했다:

- 결정 대기 중 시간이 흐른다 — 압박수가 조여오고, 게이지가 차고, 뭉개면 뺏긴다.
- 팀이 축구를 한다 — 풀백 오버랩·윙어 라인 어깨·ST 체크런, 관성 움직임, 몸 방향.
- 볼이 물리를 가진다 — 거리 비례 비행, 달리는 선수 발 앞 리드, 볼 마중.
- 조작면은 수렴했다 — 클릭=패스, Space=슬로우(1회), 운반=유인 몰기, 슛.

**이 하이브리드가 이 게임의 고유성이다**: 반사신경 게임이 아니라(시계는 초 단위로
관대), FM류 관전도 아니라(모든 패스를 내가 찬다), "압박 속에서 몇 초 안에 읽고
결정하는 축구". 유인–3자 콤비가 그 정점의 표현 도구다.

### 강점 (지킬 것)
1. **측정 문화** — 모든 변경이 게이트를 동반. 감·논쟁이 아니라 프로브가 결정.
2. **결정의 게임** — 위험·유인·수비 5택이 전부 읽고-판단하는 재미. 실시간은
   압박이지 조작 스킬이 아니다.
3. **메타→체감 글루** — 업그레이드가 숫자가 아니라 traits로 플레이에 스민다.
4. **엔진 계약** — 실시간도 신규 모듈(realtime.js)로 격리, 코어 침습 최소.

### 약점 (보완할 것)
1. **실시간 레이어의 밸런스 사각지대** — career-sim/selfplay는 시계·드리프트·런을
   안 돌린다. 계약 테스트(15어서션)는 있지만 "사람 페이스 실플레이의 goal%/붕괴율"
   을 재는 게이트가 없다. 직접 플레이 5판에서 이미 증명됐다: 시계 유무로 전승↔실점.
2. **온보딩 부채** — 튜토리얼/코치가 아직 구 문법 잔재를 가르친다. 새 동사(시계·
   Space 슬로우·운반=유인·자동 드리프트)를 가르치는 첫 경험이 없다. 하이브리드는
   낯설어서, 가르치지 않으면 혼란이다.
3. **공수 비대칭** — 실시간 호흡은 우리 공격 대기에만 있다. 수비 국면(5택)은 여전히
   완전 턴제 — 같은 경기에서 두 문법이 공존해 이질감.
4. **유인 콤비 발견성** — 정합성·시각화는 완성됐지만 "마커에게 들이대라"를 흐름
   속에서 가르치는 장치가 없고, 유인 실패(안 물음)의 피드백이 약하다(F2).
5. **소리 없음** — 킥음·관중·골 함성 부재. 현실감의 절반은 귀로 온다.
6. **런의 하드코딩** — 역할 런이 4-3-3 가정의 코드 상수다. 클럽 전술 선택이 런
   모양을 바꾸지 못한다(메타→체감의 남은 빈 축).
7. **성능 관찰 필요** — 렌더 루프가 매 프레임 evaluateBoard/preview를 돌린다.
   실시간으로 위치가 매 프레임 변하니 데스크톱은 여유지만 저사양·모바일 검증 필요.
8. **잔여 밸런스** — D4(강도 램프 man/loss 비단조), concede 패시브 3셀 미달,
   A3(후진 타이브레이크). 전부 수용 가능 수준으로 문서화됨.

---

## 1. 방향 원칙 (변경 불가 헌법)

- **P1. 결정의 게임을 지킨다.** 시계·실시간은 긴장을 만들지, 손 빠르기를 시험하지
  않는다. 슬로우(조준·Space)가 있는 이유. 튜닝은 항상 "읽을 시간은 준다" 쪽으로.
- **P2. 게이트 없는 밸런스 변경 없음.** 실시간 레이어 포함 — 사람-페이스 시뮬
  게이트를 만들어 사각지대를 없앤다(A1).
- **P3. 코어는 모듈로 감싼다.** realtime.js 패턴 유지. 엔진 수정은 해소부·계약이
  명확한 지점만.
- **P4. 유인–3자 콤비가 정점 표현이다.** 모든 신규 시스템은 이 콤비를 더 읽히게
  하거나 더 보상하게 해야지, 묻히게 하면 안 된다.

## 2. 플랜

### Phase A — 하이브리드의 완성 (정합·가독·검증) ← 최우선
> "새 정체성이 스스로 서게 한다." 신규 기능보다 지금 있는 것의 완성.

- **A1. 실시간 자기대국 게이트** — 사람 페이스(결정당 1~2.5s) 봇이 시계·드리프트·
  런·리드가 켜진 채 N경기 플레이 → goal%·붕괴율·유인 발동률 밴드를 게이트화.
  (직접 플레이 하니스를 스크립트로 승격. rAF 없이 프레임 펌핑.)
- **A2. 온보딩 리라이트** — 첫 경기 코치 3步를 새 문법으로: ①클릭=패스, 시간이
  흐른다(게이지 시계) ②압박이 붙으면? Space 슬로우로 읽기 ③운반=유인 몰기 →
  릴리스(콤비 완성 체험을 튜토리얼 안에서 1회 보장). F2(유인 실패 피드백: "마커가
  안 물었다" 시각 펄스) 포함.
- **A3. 수비 국면 실시간 호흡** — 상대 빌드업 스텝이 실시간으로 흐르고(상대 캐리
  드리프트·우리 블록 호흡), 5택 결정 순간 슬로우. 공수 문법 통일. (엔진 수비 해소
  로직은 불변 — 표현 레이어만. realtime.js 확장.)
- **A4. 사운드 패스 1차** — 킥(패스 강도별)·호루라기·골 함성·관중 베드·게이지 고압
  하트비트. Web Audio, 에셋 최소(신스 가능). 현실감 체감 대비 비용 최소인 투자.
- **A5. 게이지 위험 가독성** — 80+에서 화면 가장자리 붉은 펄스, 압박수 링 경고.
  "왜 뺏겼는지"가 항상 설명 가능하게.

### Phase B — 깊이 (콘텐츠·메타 연결)
> 완성된 문법 위에 폭을 얹는다.

- **B1. 런 프로파일 데이터화** — 역할 런 상수를 셋업(포메이션·지침)별 데이터로.
  클럽에서 "오버랩 적극/윙어 인버트/ST 타깃맨" 같은 선택이 실제 런 모양을 바꾼다
  = 메타→체감의 새 축. (formations.js에 runProfile 추가, realtime.js가 읽음.)
- **B2. 선수 개성 심화** — pace는 이미 체감. longPass→리드 거리·로빙 정확,
  pressResistance→드리프트 배짱(압박 근접 허용치), jumpiness→수비 런. 스카우팅
  문구가 실플레이 차이로 검증되게.
- **B3. 시즌 서사** — 라이벌전(상대 성향 고정+보너스), 컵 토너먼트(단판 고스테이크),
  시즌 목표/보드 기대치. 기존 지휘자·성향 시스템 재활용으로 비용 절감.
- **B4. 잔여 밸런스 정리** — D4 비단조(패시브 스킴 aggressive 피크) 재측정·수용
  문서화 or 수리, concede 패시브 3셀, A3 타이브레이크. 실시간 노브 최종 튜닝(A1
  게이트 위에서).

### Phase C — 도달 (플랫폼·공유)
> 사람들 손에 쥐어준다.

- **C1. 모바일 입력** — 터치=패스, 길게 누르기=슬로우, 운반은 홀더 드래그. 뷰포트
  반응형은 상당 부분 존재 — 입력·성능 검증이 본체.
- **C2. 성능 예산** — 프레임당 evaluateBoard 스로틀(120ms 캐시), 저사양 모드
  (노치·그림자 off). 목표: 모바일 60fps.
- **C3. 리플레이/하이라이트** — 골 장면 최근 N초 재생(위치 로그 기반), 공유 이미지.
  "내 유인 콤비 골"을 자랑할 수 있게 — 콤비 발견성의 사회적 해법.
- **C4. 배포·수집** — 정적 호스팅 + 기존 analytics로 퍼널(튜토리얼 이탈·콤비 발동률·
  평균 결정 시간) 관찰 → 튜닝 루프.

### 명시적 비목표 (지금은 안 한다)
- 실시간 조작(드리블 조종, 수비수 직접 컨트롤) — P1 위반, 다른 게임이 됨.
- 멀티플레이 — 시계 동기화·치팅 비용이 현 단계 가치 대비 과도.
- ~~3D/에셋 아트 대개편~~ → **2026-07-08 사용자 지시로 오버라이드**: 표현 기반을
  원근 카메라 3D로 전환(camera.js + 렌더러 투영화 + 미니어처 피규어). 로직·밸런스는
  평면 불변(게이트 무접촉) — "에셋 아트 대개편 없이"라는 절반은 유지(의존성 제로,
  절차 셰이딩).

## 3. 리스크와 대응

| 리스크 | 신호 | 대응 |
|---|---|---|
| 하이브리드가 낯설어 이탈 | 튜토리얼 완주율↓ | A2 최우선, C4 퍼널로 측정 |
| 시계가 스트레스로 변질 | "쫓긴다"는 피드백 | P1 — 노브는 관대 쪽으로, Space 충전 방식 재고(예: 페이즈당 1회) |
| 실시간 밸런스 회귀 | A1 게이트 이탈 | 게이트가 잡는다 — A1이 선행돼야 하는 이유 |
| 수비 실시간(A3)의 복잡도 폭발 | 구현 지연 | 표현 레이어 한정 원칙 — 엔진 해소 불변, 안 되면 슬로우 연출만 |
| 성능(모바일) | 프레임 드랍 | C2 예산 작업, 저사양 모드 |

## 4. 권장 착수 순서 (요약)

```
A1 실시간 게이트 → A2 온보딩 → A5 게이지 가독 → A4 사운드 1차 → A3 수비 호흡
→ B1 런 프로파일 → B2 선수 개성 → B4 밸런스 정리 → B3 시즌 서사
→ C2 성능 → C1 모바일 → C3 리플레이 → C4 배포
```

첫 수는 **A1(실시간 자기대국 게이트)** — 이후 모든 튜닝·기능의 안전망이며,
이 프로젝트가 지금까지 이겨온 방식(측정이 결정한다) 그대로다.
